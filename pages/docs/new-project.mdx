Um ein Projekt anzulegen benötigst Du zunächst einen Stadtpuls-Account (siehe Doku links). Du hast schon einen? Wunderbar! 
Du kannst Projekte auf Stadtpuls erstellen und verwalten und für andere Nutzer:innen der Seite öffentlich sichtbar machen. Jedes Projekt erscheint auf der <a href='/projects'>Projekte-Seite</a>.

## In drei Schritten zum neuen Projekt
1. Klicke oben rechts in der Menüleiste auf deinen **Nutzernamen** und schließlich auf **Neues Projekt**. Du siehst nun bereits die Eingabemaske für Dein neues Projekt. 
   <figure>
  <img style={{maxHeight: 400, border: '1px solid rgba(0, 0, 0, 0.05)'}}
    src="/images/projectNew.png"
    alt="Eingabemaske Neues Projekt"
  />
    <figcaption>
    Auf dieser Seite kann man ein neues Projekt anlegen
  </figcaption>
  </figure>
2. Gib alle Informationen rund um Dein Projekt ein:
- gib Deine Projekt einen aussagekräftigen Namen
- ordne ihm eine Kategorie zu, damit andere Nutzer:innen Dein Projekt besser finden können
- füge eine kurze Beschrebiung hinzu (Warum erhebst du Daten? Welche Daten werden erhoben?)
- teile uns den Hauptstadnort deiner Sensoren mit
- wähle aus, wie du künftig Sensoren zu Deinen Projekt hinzufügen möchtest
3. Klicke auf **Weiter** sobald du alle Informationen eingetragen hast

Hurray! Dein Projekt wurde erfolgreich auf Stadtpuls angelegt.

Allerdings: Dein Projekt muss noch authentifiziert werden, damit die Messwerte Deiner Sensoren künftig in unsere Datenbank geschrieben werden können.

## Projekt-Authentifizierung
Um Dein Projekt zu authentifizieren gibt es je Projekt einen sogenannten **Token**. Die Token sorgt dafür, dass alle Sensoren, die Du künftig in Deinem Projekt registrierst autorisiert sind, ihre Messwerte in unserem Datenbank zu schreiben. 

&#x26a0; Der Token ist nur einmalig und nur für Dich kurz nach erstellen Deines Projektes sichtbar! Kopiere ihn dir daher in Deine Ablage (Strg+C) oder an einen Ort Deiner Wahl (Word-Dokument, Notizen etc.). 

Wenn du Den Token erfolgreich kopiert hast, dann folge der Anleitung <a href="/docs/ttn-configuration">Applikation auf TTN konfigurieren</a>.
